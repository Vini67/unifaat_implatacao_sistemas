## Desafio 3: Inteligência e Saúde

### Requisitos de Healthcheck

**Para o banco (db):**
- Comando: Verificar se MariaDB aceita conexões
- Intervalo: 10 segundos
- Grace period: 30 segundos

**Para as aplicações (app1, app2):**
- Comando: curl na porta local
- Dependência: só sobem se db estiver healthy
- Timeout: 5 segundos

**Para o proxy:**
- Comando: verificar endpoint /health
- Dependência: só sobe se apps estiverem healthy
- Retries: 3 tentativas

### Dicas

- MariaDB usa `mysqladmin ping` para healthcheck
- Lembre-se do `condition: service_healthy`
- Configure `start_period` adequadamente

### Validação

- `docker compose ps` deve mostrar status `(healthy)`
- Parar db deve impedir apps de iniciarem
- Parar app deve impedir proxy de iniciar

## Troubleshooting Comum

### Problema: "Port already in use"

**Sintomas:**
```
Error starting userland proxy: listen tcp4 0.0.0.0:8080: bind: address already in use
```

**Diagnóstico:**
```bash
# Verificar o que está usando a porta
sudo lsof -i :8080
# ou
sudo netstat -tulpn | grep 8080
```

**Soluções:**
- Alterar porta no docker-compose.yml
- Parar processo conflitante
- Usar porta diferente (8081, 8082, etc.)



### Problema: "YAML parse error"

**Sintomas:**
```
yaml: line 15: mapping values are not allowed in this context
```

**Diagnóstico:**
```bash
# Validar sintaxe YAML
docker compose config
```

**Soluções comuns:**
- Verificar identação (sempre 2 espaços)
- Não misturar tabs e espaços
- Verificar dois pontos após chaves
- Usar aspas em valores com caracteres especiais



### Problema: "Volume mount failed"

**Sintomas:**
```
Error response from daemon: invalid mount config for type "bind": bind source path does not exist
```

**Diagnóstico:**
```bash
# Verificar se arquivo/pasta existe
ls -la nginx.conf
pwd  # Confirmar diretório atual
```

**Soluções:**
- Criar arquivo antes de montar
- Usar caminhos absolutos
- Verificar permissões de arquivo



### Problema: "Upstream not found"

**Sintomas:**
```
nginx: [emerg] host not found in upstream "app1" in /etc/nginx/nginx.conf
```

**Diagnóstico:**
```bash
# Verificar se containers estão na mesma rede
docker network ls
docker network inspect NETWORK_NAME
```

**Soluções:**
- Verificar nomes dos serviços no compose
- Confirmar que containers estão na mesma rede
- Aguardar containers iniciarem completamente



### Problema: "Healthcheck sempre failing"

**Sintomas:**
```
Container status: Up 2 minutes (unhealthy)
```

**Diagnóstico:**
```bash
# Testar comando manualmente
docker exec CONTAINER_NAME curl -f http://localhost
# Ver logs de healthcheck
docker inspect CONTAINER_NAME --format='{{json .State.Health}}'
```

**Soluções:**
- Verificar se comando funciona dentro do container
- Ajustar timeout e retries
- Aumentar start_period para apps lentas
