## 📋 Objetivo

Demonstrar domínio de provisionamento de instâncias EC2, configuração de segurança de rede e implementação de aplicações web na nuvem.

---

## 🎯 Tarefa

Criar uma infraestrutura completa de aplicação web usando EC2 com múltiplas instâncias, load balancer manual e configurações avançadas de segurança.

---

## 📝 Especificações

### Infraestrutura Multi-Instância
Implementar:
- **3 instâncias EC2** em diferentes AZs
- **Security Groups** em camadas
- **Application Load Balancer** manual (Nginx)
- **Aplicação web** distribuída
- **Monitoramento** básico

### Requisitos Técnicos

**1. Instâncias EC2:**
- 1 instância para Load Balancer (t2.micro)
- 2 instâncias para aplicação web (t2.micro)
- Todas em subnets diferentes (Multi-AZ)
- Key pairs únicos por função
- Tags organizacionais

**2. Segurança de Rede:**
- Security Group para Load Balancer (porta 80/443)
- Security Group para Web Servers (porta 80 apenas do LB)
- Security Group para SSH (apenas seu IP)
- NACLs customizadas (opcional)

**3. Aplicação Web:**
- Site de portfólio pessoal/profissional
- Páginas: Home, Sobre, Projetos, Contato
- Backend simples para formulário de contato
- Logs centralizados

**4. Load Balancer Manual:**
- Nginx configurado como proxy reverso
- Health checks personalizados
- Distribuição round-robin
- Logs detalhados

## 🔧 Scripts de Provisionamento

### provision-instances.sh
```bash
#!/bin/bash
set -e

echo "🚀 Provisionando instâncias EC2..."

# Variáveis
KEY_NAME="unifaat-portfolio-key"
AMI_ID="ami-0c02fb55956c7d316"  # Amazon Linux 2023
INSTANCE_TYPE="t2.micro"

# Criar Key Pair
echo "🔑 Criando Key Pair..."
aws ec2 create-key-pair \
    --key-name $KEY_NAME \
    --query 'KeyMaterial' \
    --output text > ${KEY_NAME}.pem
chmod 400 ${KEY_NAME}.pem

# Criar Security Groups
echo "🛡️ Criando Security Groups..."

# Load Balancer SG
LB_SG_ID=$(aws ec2 create-security-group \
    --group-name "Portfolio-LB-SG" \
    --description "Security Group para Load Balancer" \
    --query 'GroupId' --output text)

aws ec2 authorize-security-group-ingress \
    --group-id $LB_SG_ID \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0

aws ec2 authorize-security-group-ingress \
    --group-id $LB_SG_ID \
    --protocol tcp \
    --port 443 \
    --cidr 0.0.0.0/0

# Web Servers SG
WEB_SG_ID=$(aws ec2 create-security-group \
    --group-name "Portfolio-Web-SG" \
    --description "Security Group para Web Servers" \
    --query 'GroupId' --output text)

aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG_ID \
    --protocol tcp \
    --port 80 \
    --source-group $LB_SG_ID

# SSH SG
SSH_SG_ID=$(aws ec2 create-security-group \
    --group-name "Portfolio-SSH-SG" \
    --description "Security Group para SSH" \
    --query 'GroupId' --output text)

MY_IP=$(curl -s https://checkip.amazonaws.com)
aws ec2 authorize-security-group-ingress \
    --group-id $SSH_SG_ID \
    --protocol tcp \
    --port 22 \
    --cidr ${MY_IP}/32

# Lançar Load Balancer
echo "⚖️ Lançando Load Balancer..."
LB_INSTANCE_ID=$(aws ec2 run-instances \
    --image-id $AMI_ID \
    --count 1 \
    --instance-type $INSTANCE_TYPE \
    --key-name $KEY_NAME \
    --security-group-ids $LB_SG_ID $SSH_SG_ID \
    --subnet-id subnet-12345678 \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Portfolio-LoadBalancer},{Key=Role,Value=LoadBalancer},{Key=Project,Value=Portfolio}]' \
    --query 'Instances[0].InstanceId' --output text)

# Lançar Web Server 1
echo "🌐 Lançando Web Server 1..."
WEB1_INSTANCE_ID=$(aws ec2 run-instances \
    --image-id $AMI_ID \
    --count 1 \
    --instance-type $INSTANCE_TYPE \
    --key-name $KEY_NAME \
    --security-group-ids $WEB_SG_ID $SSH_SG_ID \
    --subnet-id subnet-87654321 \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Portfolio-Web-1},{Key=Role,Value=WebServer},{Key=Project,Value=Portfolio}]' \
    --query 'Instances[0].InstanceId' --output text)

# Lançar Web Server 2
echo "🌐 Lançando Web Server 2..."
WEB2_INSTANCE_ID=$(aws ec2 run-instances \
    --image-id $AMI_ID \
    --count 1 \
    --instance-type $INSTANCE_TYPE \
    --key-name $KEY_NAME \
    --security-group-ids $WEB_SG_ID $SSH_SG_ID \
    --subnet-id subnet-11223344 \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Portfolio-Web-2},{Key=Role,Value=WebServer},{Key=Project,Value=Portfolio}]' \
    --query 'Instances[0].InstanceId' --output text)

echo "✅ Instâncias provisionadas:"
echo "Load Balancer: $LB_INSTANCE_ID"
echo "Web Server 1: $WEB1_INSTANCE_ID"
echo "Web Server 2: $WEB2_INSTANCE_ID"

# Aguardar instâncias ficarem running
echo "⏳ Aguardando instâncias ficarem running..."
aws ec2 wait instance-running --instance-ids $LB_INSTANCE_ID $WEB1_INSTANCE_ID $WEB2_INSTANCE_ID

echo "🎉 Provisionamento concluído!"
```

## 🌐 Configuração do Load Balancer

### load-balancer.conf
```nginx
events {
    worker_connections 1024;
}

http {
    # Log format com informações detalhadas
    log_format detailed '$remote_addr - $remote_user [$time_local] '
                       '"$request" $status $body_bytes_sent '
                       '"$http_referer" "$http_user_agent" '
                       'upstream_addr: $upstream_addr '
                       'upstream_response_time: $upstream_response_time';

    # Upstream para web servers
    upstream portfolio_backend {
        server 10.0.1.100:80 max_fails=3 fail_timeout=30s;
        server 10.0.2.100:80 max_fails=3 fail_timeout=30s;
    }

    server {
        listen 80;
        server_name _;

        access_log /var/log/nginx/portfolio_access.log detailed;
        error_log /var/log/nginx/portfolio_error.log;

        # Health check endpoint
        location /health {
            access_log off;
            return 200 "Load Balancer OK\n";
            add_header Content-Type text/plain;
        }

        # Proxy para aplicação
        location / {
            proxy_pass http://portfolio_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # Timeouts
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
        }

        # Status page
        location /nginx-status {
            stub_status on;
            access_log off;
            allow 10.0.0.0/8;
            deny all;
        }
    }
}
```
