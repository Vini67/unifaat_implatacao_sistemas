## Passo 2: O Firewall do Banco (Security Group)

### Conceito de Encadeamento de Segurança

**Problema tradicional:**
- Liberar porta 3306 para IPs específicos
- IPs podem mudar (EC2 reinicia)
- Difícil de gerenciar em escala

**Solução AWS:**
- Security Groups podem referenciar outros Security Groups
- **Encadeamento**: SG-RDS permite acesso apenas de SG-Web-Server
- **Dinâmico**: Funciona independente de IPs

### Configurando o Security Group do RDS

**1. Aguardar criação do RDS:**
- Status deve estar: **Available**

**2. Acessar Security Groups:**
- Console EC2 → Security Groups
- Procure por: `SG-RDS-Database`

**3. Editar Inbound Rules:**
- Selecione `SG-RDS-Database` → Inbound rules → Edit inbound rules

**4. Adicionar Regra Crucial:**
- Type: **MySQL/Aurora**
- Protocol: **TCP**
- Port range: **3306**
- Source: **Custom**
  - **⚠️ IMPORTANTE**: Não digite IP!
  - Digite: `sg-` e selecione o **Web-Server-SG**
  - Ou cole o Security Group ID da EC2
- Description: `MySQL access from Web Server SG`

**5. Save rules**

### 🧠 Entendendo o Encadeamento

```
Internet → [Web-Server-SG:80] → EC2 → [SG-RDS-Database:3306] → RDS
```

**Fluxo de segurança:**
1. **Internet** acessa EC2 via porta 80 (HTTP)
2. **EC2** (com Web-Server-SG) acessa RDS via porta 3306
3. **RDS** só aceita conexões de instâncias com Web-Server-SG
4. **Isolamento**: RDS inacessível diretamente da internet

### Verificando via CLI

```bash
# Obter ID do Security Group da EC2
WEB_SG_ID=$(aws ec2 describe-security-groups --group-names "Web-Server-SG" --query 'SecurityGroups[0].GroupId' --output text)

# Obter ID do Security Group do RDS
RDS_SG_ID=$(aws ec2 describe-security-groups --filters "Name=group-name,Values=SG-RDS-Database" --query 'SecurityGroups[0].GroupId' --output text)

# Verificar regras do RDS SG
aws ec2 describe-security-groups --group-ids $RDS_SG_ID --query 'SecurityGroups[0].IpPermissions'
```

### 🧠 Endpoint vs IP Fixo

| Aspecto | Endpoint (DNS) | IP Fixo |
||-||
| **Formato** | `db.xxxxx.region.rds.amazonaws.com` | `172.31.x.x` |

## 🆘 Troubleshooting

### "Connection timed out"

**Problema:** Não consegue conectar no RDS  
**Causa mais comum:** Security Group mal configurado

**Diagnóstico:**
```bash
# Verificar se EC2 tem o Security Group correto
aws ec2 describe-instances --filters "Name=instance-state-name,Values=running" --query 'Reservations[0].Instances[0].SecurityGroups'

# Verificar regras do RDS Security Group
aws ec2 describe-security-groups --filters "Name=group-name,Values=SG-RDS-Database" --query 'SecurityGroups[0].IpPermissions'
```

**Soluções:**
1. Verificar se RDS SG permite porta 3306 do Web-Server-SG
2. Confirmar que EC2 está usando Web-Server-SG
3. Aguardar alguns minutos após mudanças de SG

### "Access denied for user"

**Problema:** Credenciais incorretas  
**Soluções:**
- Verificar username: `admin`
- Verificar senha configurada na criação
- Tentar resetar senha via console RDS

### "Unknown database"

**Problema:** Banco `unifaat_db` não existe  
**Soluções:**
```sql
-- Listar bancos disponíveis
SHOW DATABASES;

-- Criar banco se não existir
CREATE DATABASE unifaat_db;
USE unifaat_db;
```

### RDS não aparece como "Available"

**Problema:** Status fica em "Creating" por muito tempo  
**Soluções:**
- Aguardar até 20 minutos (normal)
- Verificar se região está correta
- Verificar limites da conta Free Tier



## 🧹 Alerta de Encerramento

### ⚠️ IMPORTANTE: Free Tier Limits

**RDS Free Tier:**
- **750 horas/mês** de db.t2.micro ou db.t3.micro
- **20GB** de armazenamento
- **20GB** de backup

**⚠️ CUIDADO:**
- Instância RDS **sempre consome horas**, mesmo parada
- **Única forma de parar cobrança**: Deletar instância
- Snapshots **continuam cobrando** armazenamento

### Procedimento de Limpeza

**1. Fazer backup final (opcional):**
```bash
# Criar snapshot antes de deletar
aws rds create-db-snapshot --db-instance-identifier unifaat-ads-database --db-snapshot-identifier unifaat-final-backup
```

**2. Deletar instância RDS:**
```bash
# Via CLI (sem snapshot final)
aws rds delete-db-instance --db-instance-identifier unifaat-ads-database --skip-final-snapshot

# Via Console
# RDS → Databases → unifaat-ads-database → Actions → Delete
# ❌ Create final snapshot? (para economizar)
# Digite: delete me
# Delete
```

**3. Deletar Security Group:**
```bash
# Aguardar RDS ser deletado completamente, depois:
aws ec2 delete-security-group --group-name "SG-RDS-Database"
```

**4. Deletar snapshots manuais:**
```bash
# Listar snapshots
aws rds describe-db-snapshots --snapshot-type manual

# Deletar snapshot específico
aws rds delete-db-snapshot --db-snapshot-identifier SNAPSHOT-ID
```

## 🎓 Conexão com Arquiteturas Reais

**Padrões de arquitetura:**

- **3-Tier Architecture**: Web → App → Database
- **Microservices**: Cada serviço com seu banco
- **Read/Write Split**: Master para escrita, replicas para leitura
- **Database per Service**: Isolamento de dados por domínio

**Integração com outros serviços:**
- **Lambda**: Funções serverless acessando RDS
- **ECS/EKS**: Containers conectando via Service Discovery
- **ElastiCache**: Cache em memória para performance
- **CloudWatch**: Monitoramento de métricas de banco
