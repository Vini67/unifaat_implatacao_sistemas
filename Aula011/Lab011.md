## Passo 1: Criação do Bucket

### Regras Fundamentais do S3

**Nome Global Único:**
- Buckets têm **namespace global** (único em toda AWS)
- Formato DNS válido (sem espaços, maiúsculas, caracteres especiais)
- Entre 3-63 caracteres

**Escolha da Região:**
- **Latência**: Mais próximo dos usuários
- **Compliance**: Dados ficam na região escolhida
- **Custos**: Regiões têm preços diferentes

### Criando o Bucket via Console

**1. Acesse S3:**
- Console AWS → S3 → Create bucket

**2. Configurações Básicas:**
- Bucket name: `unifaat-ads-site-[SEU-RA]` (ex: unifaat-ads-site-123456)
- AWS Region: **us-east-1** (mais barato) ou **sa-east-1** (menor latência)

**3. Object Ownership:**
- **ACLs disabled** (recomendado - use bucket policies)

**4. Block Public Access:**
- **Mantenha TODAS marcadas** por enquanto (configuraremos depois)

**5. Bucket Versioning:**
- **Disable** (para economizar custos no lab)

**6. Tags (Opcional):**
- Key: `Environment`, Value: `Learning`
- Key: `Course`, Value: `ImplementacaoSoftware`

**7. Default Encryption:**
- **Amazon S3 managed keys (SSE-S3)** (padrão)

**8. Create Bucket:**
- Review e Create bucket

### 🧠 Por que Nome Global Único?

**Namespace DNS:**
- Buckets são acessíveis via URLs: `https://bucket-name.s3.region.amazonaws.com`
- DNS não permite duplicatas globais
- Facilita CDN e integração com outros serviços

**Exemplo de URLs:**
```
https://unifaat-ads-site-123456.s3.us-east-1.amazonaws.com
https://unifaat-ads-site-123456.s3-website-us-east-1.amazonaws.com (website)
```



## Passo 2: Segurança e Acesso Público

### Entendendo Block Public Access

**Por padrão, S3 bloqueia acesso público** para prevenir vazamentos acidentais de dados.

### Desabilitando Block Public Access

**⚠️ ATENÇÃO: Faça isso APENAS para sites públicos!**

**1. Selecionar Bucket:**
- S3 → Buckets → Clique no seu bucket

**2. Permissions Tab:**
- Clique na aba **Permissions**

**3. Block Public Access:**
- Clique em **Edit**
- **Desmarque TODAS** as opções:
  - ❌ Block all public access
  - ❌ Block public access to buckets and objects granted through new access control lists (ACLs)
  - ❌ Block public access to buckets and objects granted through any access control lists (ACLs)
  - ❌ Block public access to buckets and objects granted through new public bucket or access point policies
  - ❌ Block public access to buckets and objects granted through any public bucket or access point policies

**4. Confirmar:**
- Digite: `confirm`
- Save changes

### Configurando Bucket Policy

**1. Bucket Policy:**
- Ainda na aba Permissions → Bucket policy → Edit

**2. Cole a seguinte política JSON:**

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::unifaat-ads-site-SEU-RA/*"
        }
    ]
}
```

**⚠️ IMPORTANTE:** Substitua `unifaat-ads-site-SEU-RA` pelo nome real do seu bucket!

**3. Save changes**

### 🔐 Entendendo a Política

**Componentes da política:**
- **Version**: Versão da linguagem de política
- **Statement**: Declaração de permissão
- **Sid**: Identificador da declaração (opcional)
- **Effect**: Allow ou Deny
- **Principal**: Quem pode executar (*)= todos)
- **Action**: O que pode fazer (s3:GetObject = ler objetos)
- **Resource**: Em quais recursos (ARN do bucket/*)

**Por que apenas GetObject?**
- **Segurança**: Permite apenas leitura, não escrita
- **Princípio do menor privilégio**: Apenas o necessário
- **Proteção**: Evita upload malicioso

## Passo 4: Gestão de Arquivos via CLI

### Comandos Essenciais do S3

### Upload de Arquivo Individual

```bash
# Fazer upload do index.html
aws s3 cp website/index.html s3://unifaat-ads-site-SEU-RA/

# Fazer upload do error.html
aws s3 cp website/error.html s3://unifaat-ads-site-SEU-RA/
```

### Sincronização de Pasta Completa

```bash
# Sincronizar toda a pasta website
aws s3 sync website/ s3://unifaat-ads-site-SEU-RA/

# Sincronizar com delete (remove arquivos que não existem localmente)
aws s3 sync website/ s3://unifaat-ads-site-SEU-RA/ --delete
```

### Listagem de Conteúdo

```bash
# Listar arquivos no bucket
aws s3 ls s3://unifaat-ads-site-SEU-RA/

# Listar com detalhes (tamanho, data)
aws s3 ls s3://unifaat-ads-site-SEU-RA/ --human-readable --summarize

# Listar recursivamente (se houver subpastas)
aws s3 ls s3://unifaat-ads-site-SEU-RA/ --recursive
```

### Comandos Avançados

```bash
# Copiar com metadados específicos
aws s3 cp website/index.html s3://unifaat-ads-site-SEU-RA/ --metadata "author=UniFAAT,version=1.0"

# Definir Content-Type específico
aws s3 cp website/style.css s3://unifaat-ads-site-SEU-RA/ --content-type "text/css"

# Upload com cache control
aws s3 cp website/logo.png s3://unifaat-ads-site-SEU-RA/ --cache-control "max-age=86400"
```

### 🧠 Diferença: cp vs sync

| Comando | Comportamento | Uso |
|||--|
| **aws s3 cp** | Copia arquivo específico | Upload individual |
| **aws s3 sync** | Sincroniza diretórios | Deploy completo |
| **sync --delete** | Remove arquivos extras | Limpeza automática |



## Passo 5: Teste de Resiliência

### Obtendo o Website Endpoint

**1. No Console S3:**
- Seu bucket → Properties → Static website hosting
- Copie o **Bucket website endpoint**

**Formato típico:**
```
http://unifaat-ads-site-123456.s3-website-us-east-1.amazonaws.com
```

### 🧠 Diferença: ARN vs Endpoint URL

| Tipo | Formato | Uso |
|||--|
| **S3 ARN** | `arn:aws:s3:::bucket-name/*` | Políticas IAM/Bucket |
| **S3 API URL** | `https://bucket.s3.region.amazonaws.com` | API REST |
| **Website URL** | `http://bucket.s3-website-region.amazonaws.com` | Navegador |

## 💰 Atenção: Custos do S3

### Modelo de Cobrança

**Armazenamento:**
- **Standard**: $0.023 por GB/mês (primeiros 50TB)
- **Exemplo**: 1GB = ~$0.02/mês

**Requisições:**
- **PUT/POST**: $0.0005 por 1.000 requests
- **GET**: $0.0004 por 1.000 requests
- **Exemplo**: 10.000 pageviews = ~$0.004

**Transferência de Dados:**
- **Upload**: Gratuito
- **Download**: Primeiros 100GB/mês gratuitos
- **Depois**: $0.09 por GB

### Calculadora de Custos

**Site pequeno (exemplo):**
- 100MB de arquivos
- 1.000 pageviews/mês
- **Custo mensal**: ~$0.10 (10 centavos!)

**Comparação:**
- **VPS básico**: $5-20/mês
- **S3 Static**: $0.10/mês
- **Economia**: 98%+

## 🧹 Procedimento de Limpeza

### ⚠️ IMPORTANTE: Esvaziar Antes de Deletar

**S3 não permite deletar buckets com conteúdo!**

### Limpeza via Console

**1. Esvaziar Bucket:**
- S3 → Seu bucket → **Empty**
- Digite: `permanently delete`
- Empty

**2. Deletar Bucket:**
- S3 → Seu bucket → **Delete**
- Digite o nome do bucket
- Delete bucket

### Limpeza via CLI

```bash
# Remover todos os arquivos
aws s3 rm s3://unifaat-ads-site-SEU-RA/ --recursive

# Deletar bucket
aws s3 rb s3://unifaat-ads-site-SEU-RA/

# Comando combinado (cuidado!)
aws s3 rb s3://unifaat-ads-site-SEU-RA/ --force
```



## 📚 Comandos de Referência Rápida

```bash
# Gestão de Buckets
aws s3 mb s3://bucket-name                    # Criar bucket

# Upload/Download
aws s3 cp file.txt s3://bucket/               # Upload arquivo
aws s3 cp s3://bucket/file.txt .              # Download arquivo

# Listagem
aws s3 ls                                     # Listar buckets
aws s3 ls s3://bucket/                        # Listar objetos
aws s3 ls s3://bucket/ --recursive            # Listar recursivo

# Políticas e Configurações
aws s3api get-bucket-policy --bucket NAME     # Ver política
aws s3api get-bucket-website --bucket NAME    # Ver config website
aws s3api put-bucket-policy --bucket NAME --policy file://policy.json
```
