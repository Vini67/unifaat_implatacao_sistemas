### 🧠 Importância do Shebang (#!/bin/bash)

**#!/bin/bash** é o **shebang** - linha que define qual interpretador usar:

- **Obrigatório**: Sem ele, o script pode não executar
- **Especifica interpretador**: bash, sh, python, etc.
- **Primeira linha**: Deve ser exatamente a primeira linha
- **Portabilidade**: Garante execução consistente

**Exemplos:**
```bash
#!/bin/bash          # Bash script
#!/bin/sh            # Shell script (mais compatível)
#!/usr/bin/python3   # Python script
#!/usr/bin/env node  # Node.js script
```

## Passo 3: Criando a Golden Image (AMI)

### O que é uma Golden Image?

Uma **Golden Image** (AMI customizada) é um "snapshot" de uma instância já configurada. Permite criar novas instâncias **instantaneamente** com toda configuração pré-instalada.

### Criando a AMI

**1. Selecionar Instância:**
- EC2 → Instances → Selecione `WebServer-Automated-UniFAAT`
- **Status deve ser**: Running e página funcionando

**2. Create Image:**
- Actions → Image and templates → **Create image**

**3. Configurar AMI:**
- Image name: `WebServer-UniFAAT-Golden-v1.0`
- Image description: `Servidor web automatizado com Nginx - Curso ADS UniFAAT`
- **No reboot**: ❌ (deixe marcado para garantir consistência)

**4. Create Image:**
- Clique em **Create image**

### 🔄 O que Acontece Durante a Criação?

**Processo automático:**
1. **Reboot da instância**: Para garantir consistência do filesystem
2. **Snapshot do EBS**: Cria cópia do disco
3. **Registro da AMI**: Torna a imagem disponível
4. **Tempo**: 5-10 minutos dependendo do tamanho

**Status da instância:**
- Temporariamente: `stopping` → `stopped` → `running`
- **Normal**: Instância volta ao funcionamento

### Monitorando a Criação

```bash
# Via CLI - monitorar AMIs
aws ec2 describe-images --owners self --query 'Images[*].[Name,State,CreationDate]' --output table

# Verificar snapshots
aws ec2 describe-snapshots --owner-ids self --query 'Snapshots[*].[SnapshotId,State,Progress,StartTime]' --output table
```

## Passo 5: Troubleshooting de Automação

### Logs Essenciais para Debugging

**1. Log do Cloud-Init (Principal):**
```bash
sudo tail -f /var/log/cloud-init-output.log
```
- **Contém**: Toda saída do User Data
- **Localização**: Sempre neste arquivo
- **Uso**: Primeiro lugar para investigar falhas

**2. Log do Bootstrap (Customizado):**
```bash
sudo tail -f /var/log/bootstrap.log
```
- **Contém**: Logs específicos do nosso script
- **Vantagem**: Mais organizado e específico

**3. Log do Sistema:**
```bash
sudo journalctl -u nginx -f
```
- **Contém**: Logs específicos do Nginx
- **Uso**: Problemas com o serviço

### Problemas Comuns e Soluções

#### User Data não executa

**Sintomas:**
- Instância inicia mas script não roda
- Nginx não está instalado

**Diagnóstico:**
```bash
# Verificar se cloud-init rodou
sudo systemctl status cloud-init

# Ver logs de erro
sudo cat /var/log/cloud-init.log | grep -i error
```

**Soluções:**
- Verificar shebang (#!/bin/bash)
- Confirmar que User Data foi colado corretamente
- Verificar permissões de rede (Security Group)

#### Script falha parcialmente

**Sintomas:**
- Algumas partes funcionam, outras não
- Nginx instalado mas página não carrega

**Diagnóstico:**
```bash
# Ver onde parou
sudo cat /var/log/cloud-init-output.log | tail -50

# Verificar status dos serviços
sudo systemctl status nginx
```

**Soluções:**
- Adicionar mais logs no script
- Testar comandos individualmente
- Verificar conectividade de rede

#### AMI não funciona como esperado

**Sintomas:**
- Instância da AMI não tem configurações esperadas
- Serviços não iniciam automaticamente

**Diagnóstico:**
```bash
# Verificar se serviços estão habilitados
sudo systemctl is-enabled nginx

# Ver logs de boot
sudo journalctl -b
```

**Soluções:**
- Recriar AMI após corrigir problemas
- Verificar se serviços foram habilitados (enable)
- Testar AMI em instância limpa

## 🧹 Procedimento de Limpeza

### ⚠️ IMPORTANTE: Evitar Custos de Armazenamento

AMIs e Snapshots **geram custos** de armazenamento mesmo quando instâncias estão paradas.

### Limpeza Completa

**1. Terminar Instâncias:**
```bash
# Listar instâncias do lab
aws ec2 describe-instances --filters "Name=tag:Name,Values=WebServer-*" --query 'Reservations[*].Instances[*].[InstanceId,Tags[?Key==`Name`].Value|[0],State.Name]' --output table

# Terminar todas (substitua IDs)
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0 i-0987654321fedcba0
```

**2. Deletar AMI:**
```bash
# Listar AMIs
aws ec2 describe-images --owners self --query 'Images[*].[ImageId,Name,State]' --output table

# Deletar AMI (substitua AMI-ID)
aws ec2 deregister-image --image-id ami-1234567890abcdef0
```

**3. Deletar Snapshots:**
```bash
# Listar snapshots
aws ec2 describe-snapshots --owner-ids self --query 'Snapshots[*].[SnapshotId,Description,State]' --output table

# Deletar snapshot (substitua SNAP-ID)
aws ec2 delete-snapshot --snapshot-id snap-1234567890abcdef0
```

### Limpeza via Console

**AMI:**
- EC2 → AMIs → Selecionar AMI → Actions → Deregister AMI

**Snapshots:**
- EC2 → Snapshots → Selecionar snapshot → Actions → Delete snapshot

**Instâncias:**
- EC2 → Instances → Selecionar instâncias → Instance state → Terminate instance
