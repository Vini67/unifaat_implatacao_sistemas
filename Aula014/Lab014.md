## Passo 1: Criando o Launch Template (A Receita)

### O que é um Launch Template?

**Launch Template** é uma "receita" que define como criar instâncias EC2. Especifica AMI, tipo de instância, security groups, user data, etc. É a base para Auto Scaling Groups.

### Criando Security Group para ALB

**1. Criar Security Group para Load Balancer:**
```bash
# Criar SG para ALB
aws ec2 create-security-group --group-name "ALB-WebServer-SG" --description "Security Group para Application Load Balancer"

# Obter SG ID
ALB_SG_ID=$(aws ec2 describe-security-groups --group-names "ALB-WebServer-SG" --query 'SecurityGroups[0].GroupId' --output text)

# Permitir HTTP (80) de qualquer lugar
aws ec2 authorize-security-group-ingress --group-id $ALB_SG_ID --protocol tcp --port 80 --cidr 0.0.0.0/0
```

**2. Criar Security Group para Instâncias:**
```bash
# Criar SG para instâncias do ASG
aws ec2 create-security-group --group-name "ASG-Instance-SG" --description "Security Group para instâncias do Auto Scaling Group"

# Obter SG ID
ASG_SG_ID=$(aws ec2 describe-security-groups --group-names "ASG-Instance-SG" --query 'SecurityGroups[0].GroupId' --output text)

# Permitir HTTP (80) apenas do ALB
aws ec2 authorize-security-group-ingress --group-id $ASG_SG_ID --protocol tcp --port 80 --source-group $ALB_SG_ID

# Permitir SSH do seu IP (para troubleshooting)
MY_IP=$(curl -s https://checkip.amazonaws.com)
aws ec2 authorize-security-group-ingress --group-id $ASG_SG_ID --protocol tcp --port 22 --cidr $MY_IP/32
```

## Passo 2: O Balanceador de Carga (ALB)

### Criando Application Load Balancer

**1. Console EC2:**
- EC2 → Load Balancers → Create load balancer

**2. Escolher tipo:**
- **Application Load Balancer** → Create

**3. Configurações básicas:**
- Load balancer name: `WebServer-UniFAAT-ALB`
- Scheme: **Internet-facing**
- IP address type: **IPv4**

**4. Network mapping:**
- VPC: **Default VPC**
- Mappings: **Selecione TODAS as Availability Zones disponíveis**
  - us-east-1a, us-east-1b, us-east-1c, etc.

**5. Security groups:**
- **ALB-WebServer-SG** (criado anteriormente)

### Configurando Target Group

**6. Listeners and routing:**
- Protocol: **HTTP**
- Port: **80**
- Default action: **Create target group**

**7. Target group configuration:**
- Target type: **Instances**
- Target group name: `WebServer-UniFAAT-TG`
- Protocol: **HTTP**
- Port: **80**
- VPC: **Default VPC**

**8. Health checks:**
- Health check protocol: **HTTP**
- Health check path: **/**
- Advanced health check settings:
  - Healthy threshold: **2** (2 checks consecutivos = healthy)
  - Unhealthy threshold: **3** (3 checks consecutivos = unhealthy)
  - Timeout: **5 seconds**
  - Interval: **10 seconds**
  - Success codes: **200**

**9. Create target group**

**10. Finalizar ALB:**
- Selecione o Target Group criado
- Create load balancer

### 🧠 Entendendo Health Checks

| Parâmetro | Valor | Significado |
|--|-|-|
| **Healthy threshold** | 2 | 2 checks OK = instância healthy |
| **Unhealthy threshold** | 3 | 3 checks falha = instância unhealthy |
| **Timeout** | 5s | Tempo limite para resposta |
| **Interval** | 10s | Intervalo entre checks |
| **Success codes** | 200 | Códigos HTTP considerados sucesso |

  - Health check grace period: **300 seconds**

### Configurando Capacidade

**5. Configure group size and scaling policies:**
- Desired capacity: **2**
- Minimum capacity: **2**
- Maximum capacity: **4**

**6. Scaling policies:**
- **No scaling policies** (para simplificar o lab)

**7. Add notifications:**
- **Skip** (não configurar por enquanto)

**8. Add tags:**
- Key: `Name`, Value: `WebServer-ASG-Instance`
- Key: `Environment`, Value: `Learning`
- Key: `Course`, Value: `ImplementacaoSoftware`

**9. Create Auto Scaling group**

### 🧠 Por que ELB Health Check?

| Tipo | O que Monitora | Problema |
||-|-|
| **EC2 Health Check** | Status da instância | Detecta apenas falha de hardware |
| **ELB Health Check** | Resposta HTTP da aplicação | Detecta falha de aplicação |

**Exemplo:**
- Nginx para de funcionar mas instância continua rodando
- EC2 Health Check: ✅ Healthy (instância OK)
- ELB Health Check: ❌ Unhealthy (aplicação falhou)



## Passo 4: Teste de Resiliência (Self-Healing)

### Aguardando Inicialização

```bash
# Verificar status do ASG
aws autoscaling describe-auto-scaling-groups --auto-scaling-group-names "WebServer-UniFAAT-ASG" --query 'AutoScalingGroups[0].[MinSize,DesiredCapacity,MaxSize]' --output table

# Verificar instâncias do ASG
aws autoscaling describe-auto-scaling-instances --query 'AutoScalingInstances[?AutoScalingGroupName==`WebServer-UniFAAT-ASG`].[InstanceId,LifecycleState,AvailabilityZone]' --output table

# Verificar targets no ALB
aws elbv2 describe-target-health --target-group-arn $(aws elbv2 describe-target-groups --names "WebServer-UniFAAT-TG" --query 'TargetGroups[0].TargetGroupArn' --output text)
```

### 🎯 Desafio: Teste de Self-Healing

**1. Aguardar 2 instâncias healthy:**
- Console EC2 → Target Groups → WebServer-UniFAAT-TG
- Aguardar até 2 targets aparecerem como **healthy**

**2. Identificar instâncias do ASG:**
- Console EC2 → Instances
- Procure por instâncias com tag `Name: WebServer-ASG-Instance`

**3. Simular falha (Terminate):**
- Selecione UMA das instâncias do ASG
- Instance state → **Terminate instance**
- Confirme a terminação

**4. Observar self-healing:**
- **Imediatamente**: ASG detecta instância terminada
- **30-60s**: ASG inicia nova instância
- **2-3min**: Nova instância passa nos health checks
- **Resultado**: Sempre 2 instâncias healthy

### Monitorando o Self-Healing

```bash
# Monitorar em tempo real
watch -n 10 'aws autoscaling describe-auto-scaling-instances --query "AutoScalingInstances[?AutoScalingGroupName==\`WebServer-UniFAAT-ASG\`].[InstanceId,LifecycleState,AvailabilityZone]" --output table'

# Verificar atividade do ASG
aws autoscaling describe-scaling-activities --auto-scaling-group-name "WebServer-UniFAAT-ASG" --max-items 5
```



## Passo 5: Teste de Balanceamento

### Obtendo DNS do Load Balancer

```bash
# Obter DNS do ALB
ALB_DNS=$(aws elbv2 describe-load-balancers --names "WebServer-UniFAAT-ALB" --query 'LoadBalancers[0].DNSName' --output text)
echo "ALB DNS: $ALB_DNS"
```

### 🎯 Desafio: Visualizando o Balanceamento

**1. Acesse o DNS do ALB:**
- **NUNCA use IP das instâncias**
- **SEMPRE use DNS do Load Balancer**
- URL: `http://ALB_DNS`

**2. Teste de balanceamento:**
- Pressione **F5** (refresh) repetidamente
- Observe mudança no **Instance ID**
- Observe mudança na **Availability Zone**
- Observe mudança no **IP Privado**

**3. Teste via terminal:**
```bash
# Fazer múltiplas requisições
for i in {1..10}; do
  echo "Requisição $i:"
  curl -s http://$ALB_DNS | grep -E "(Instance ID|Availability Zone)" | head -2
  echo ""
  sleep 1
done
```

### 🧠 Conceito de Draining

**O que acontece quando instância é removida:**

1. **ALB para de enviar novas requisições**
2. **Conexões existentes continuam** (até timeout)
3. **Draining period**: 300s por padrão
4. **Após draining**: Instância é terminada

**Benefício:** Zero downtime para usuários conectados

### Procedimento de Limpeza OBRIGATÓRIO

**1. Deletar Auto Scaling Group:**
```bash
# Reduzir capacidade para 0
aws autoscaling update-auto-scaling-group --auto-scaling-group-name "WebServer-UniFAAT-ASG" --min-size 0 --desired-capacity 0

# Aguardar terminação das instâncias
aws autoscaling wait instances-terminated --auto-scaling-group-name "WebServer-UniFAAT-ASG"

# Deletar ASG
aws autoscaling delete-auto-scaling-group --auto-scaling-group-name "WebServer-UniFAAT-ASG" --force-delete
```

**2. Deletar Load Balancer:**
```bash
# Obter ARN do ALB
ALB_ARN=$(aws elbv2 describe-load-balancers --names "WebServer-UniFAAT-ALB" --query 'LoadBalancers[0].LoadBalancerArn' --output text)

# Deletar ALB
aws elbv2 delete-load-balancer --load-balancer-arn $ALB_ARN
```

**3. Deletar Target Group:**
```bash
# Aguardar ALB ser deletado (2-3 minutos)
sleep 180

# Obter ARN do Target Group
TG_ARN=$(aws elbv2 describe-target-groups --names "WebServer-UniFAAT-TG" --query 'TargetGroups[0].TargetGroupArn' --output text)

# Deletar Target Group
aws elbv2 delete-target-group --target-group-arn $TG_ARN
```

**4. Deletar Launch Template:**
```bash
aws ec2 delete-launch-template --launch-template-name "WebServer-UniFAAT-Template"
```

**5. Deletar Security Groups:**
```bash
# Deletar SG das instâncias
aws ec2 delete-security-group --group-id $ASG_SG_ID

# Deletar SG do ALB
aws ec2 delete-security-group --group-id $ALB_SG_ID
```

## 🎓 Conexão com Arquiteturas Reais

**Padrões de arquitetura:**

- **Multi-Tier**: ALB → Web Tier → App Tier → DB Tier
- **Microservices**: ALB por serviço com service discovery
- **Blue/Green**: Dois ambientes com switch via ALB
- **Canary**: Gradual rollout com weighted routing

**Integração com outros serviços:**
- **Route 53**: DNS com health checks
- **CloudFront**: CDN global com ALB como origin
- **WAF**: Web Application Firewall no ALB
- **Certificate Manager**: SSL/TLS automático
